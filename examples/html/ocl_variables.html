Variable Definition
Launch Delphi-example | Launch CBuilder-example

This example uses the unit BoldOclVariables to define a variable global_VAT. It shows how a retail price can be calculated from a net price and a VAT value.

How to run

Add products and enter a net price in the grid. Change the VAT value in the edit field at the bottom of the form and see how the value in the retail price column is changed. Changes in the net price also affects the retail price. You can't change the value of the retail price directly by editing the field. The only way to do that is to change the values of either the net price or VAT.

Use of BoldOclVariables

The BoldOclVariables unit is used whenever you want to define a variable to use in an OCL expression. In this example connect the BoldOclVariables component to the BoldProperties of column RetailPrice in the grid. The variable itself can then be used in an OCL expression to calculate a value for the column. 

The varibles defined in a BoldOclVariables is currently limited to use outside the BoldModel. If You need a global variable for use inside the model (e.g. in the Derivation OCL for a derived attribute), you need to define this in code (see the example referenced below).
