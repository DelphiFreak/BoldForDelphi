Derived Attributes
Launch Delphi-example | Launch CBuilder-example

A derived attribute is a value that doesn´t need to be persistent since it can always be calculated using other values. Examples of derived attributes are the total sum of an invoice, the full name of a person or the age of a person given the birth date.

This example shows a code free way to implement derived attributes. The derivation is defined by an OCL expression in the model. Observe that the derived attribute is write protected.

There is also a small example of how to define global variables within the OCL evaluator. This variable is used inside the OCL expression that controls the derived attribute.

How to run

The example shows how a retail price can be calculated from a net price and a VAT value.  Add products and enter net price in the grid. Change the VAT value in the edit field at the bottom of the form and see how the value in the retail price column is changed. Changes in the net price also affects the retail price. You can't change the value of the retail price directly by editing the field. The only way to do that is to change the value of either the net price or the VAT.

Derive attributes

In this example the attribute retail_Price in the Product class is derived.
