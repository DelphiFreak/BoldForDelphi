This example shows how custom attribute types are used with $(PRODUCT_BASE_NAME).

The five example attributes are (all generated using the Bold Attribute Wizard):

  • Name (TBAName)
    Adds no new data or behaviour. Subclass to TBAString.
  • ShortString (TBAShortString)
    Enforces a maximum length of 25 characters by overriding the ValidateString method. Subclass to TBAString.
  • SwedishSocSec (TBASwedishSocSec)
    Enforces the checksum rules of swedish social security numbers by overriding both the ValidateString and the ValidateCharacter mehtods. Subclass to TBAString.
  • BudgetStatus (TBABudgetStatus)
    A custom multivalued type with 3 different values ("preliminary", "current" and "old"). Subclass to TBAValueSet.
  • Coordinate (TBACoordinate)
    An attribute with an x, y and z component to store space coordinates. It will need it's own perisistence mapper since there is no other mapper that knows how to store coordinates. This is also generated by the wizard. The Coordinate attribute has 5 different string representations. The first is a composite string with all three parts. The second, third and fourth representations are the parts (x, y, z), and the fifth is a string representation of the length ofthe vector represented by the coordinate). More logic could be implemented in the coordinate attribute type, such as vector multiplication, vector addition, and various transformations.

How to run

Since the attributes are used by the RTModel in designtime to help you write OCL expressions, the attributes must be installed in designtime packages. There is a sample-package included (SampleAttrPackD5/D6.dpk). You should compile and install this package before you start the demo application.

Create custom attributes

All custom attributes must first be implemented, but you don't have to do that from scratch. There is an attribute wizard that gives you the nessecary code for most cases. You find the wizard under the Bold menu in Delphi. Once implemented you need to add the definition of the attribute in a TBoldTypeNameHandle (the wizard code will tell you what to enter in this handle).


Note! Please make sure to use the "Create DB & Alias" button before attempting to open the system for the first time. Once the database and alias is established you only have to use the "Open system" button when running the example program.
