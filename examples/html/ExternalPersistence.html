External Persistence
Launch Delphi-example for EventDriven ExternalPersistence

Launch Delphi-example for ModelDriven External Persistence

These example shows how to integrate data in external databases with data stored in a Bold-generated database.

Both examples uses the database that whips with Delphi in <program files>\common files\Borland shared\data

The event driven example uses the paradox version of this database, and the model-driven example uses the Interbase version of this database (mastsql.gdb)

If you have installed your "<common files>\borland shared" in a non standard location, you might need to change the path-settings in the database components that access the external databases.

Common behaviour

Both examples share the fact that their model contains classes, relations and attributes that already exists in an external data source. These classes/attributes/associations are tagged in the model with a tagged value called "Storage". For the classes, the value should be "PartiallyExternal", for attributes, it can be either "External" or "ExternalKey", and for associations, it should be "External".

As objects are loaded from the external datasouurce their existence will be synchronized into the bold database, but their data will not be duplicated (except for the key, which will be used to find the external object again).

The event driven example (BE_ExternalPersistence)

This example shows how to use the event driven persistence handle to access data from an external source. The persistence handle has a collection of config-items, one per class that should be handled. Each collection item has a number of events that define how  objecs of this class is retrieved and stored. The example shows how to read and write data from a database, but the external storage could be anything:

  • POP/IMAP-server
  • NNTP/news server
  • XML database
  • Flatfile
  • LDAP directory

The model driven example (BE_ExternalPersistence)

The model driven example shows how to load data from a SQL-based database, using the model as the only source of information how to map the data to and from the external database. The tagged value "Storage" defines which elements should be stored/retrieved externally, and the tagged values "TableName" (for classes) and "ColumnName" (for attributes/association ends) defines exactly where to find the data.

How to run

The "Create DB"-button in this example will only create the Bold database. it will not erase or modify the external database.

Note! Please make sure to use the "Create DB " button before attempting to open the system for the first time. Once the database and alias is established you only have to use the "Open system" button when running the example program.
