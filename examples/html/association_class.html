Association Class
Launch Delphi-example | Launch CBuilder-example

An example of how to implement association classes in $(PRODUCT_BASE_NAME). 

How to run

Add instances of persons and companies to the grids using the navigators. Select an instance of a company and then drag a person to the Employees listbox. See what happens in the other grids. Select a person and drag an instance of a company to the Employers listbox.

The BoldGrid in the center of the example window contains a list of objects of the association class Job. For each object in this list you are able to navigate through both association ends and retrieve their attributes.

Implement association classes

The model editor automatically creates an association class when importing a multi-multi relation from a Rose model. If you choose to create the model entirely inside the model editor you will have to manually create a class and assign it to the relation. In most cases you don't explicitly use the association class. If you find the need to assign attributes to a relation (in this case job descriptions on a relation between a person and his employer), these attributes will belong inside the association class.

In this example the Job class is linked to the association Job in the Rose model.


Note! Please make sure to use the "Create DB & Alias" button before attempting to open the system for the first time. Once the database and alias is established you only have to use the "Open system" button when running the example program.
