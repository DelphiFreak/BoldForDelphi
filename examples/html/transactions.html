Transactions
Launch Delphi-example | Launch CBuilder-example

This example shows how to use the in-memory transactions in $(PRODUCT_BASE_NAME). It implements a simple banking system, where the user issues a batch of requests. The different kinds of requests are:

  • Transfer money from one account to another.
  • Change the credit limit for an account.
  • Close an account.

How to run

Add a couple of accounts and create requests on them. The requests are stored and won't be runned until you explicitly ask them to by clicking the Run button.

The requests must either be carried out in total or not at all. There are different rules that may cause the transaction to fail, such as: 

  • An account cannot be closed unless the total is 0
  • At the end of a transaction no account may be below its credit limit.

Control transactions

To control the transaction of data to the database you implement the starting, committing and rolling back of transactions on the system handle. You can see how it's done in this example in the on-click event on the Run button.

Use of ReceiveQueryFromOwned

This example also make use of the overridden framework method ReceiveQueryFromOwned. Open the implementation classfile to see it.
