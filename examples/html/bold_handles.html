Bold Handles
This is a brief description of the different types of handles available in Bold. For each handle there is also one or more links to examples using the handle.

The handles are nonvisual components. The various types of handles are the developers primary interface to the elements of an objectspace.

The handles have a value which is a TBoldElement. TBoldElement is a superclass of all parts or the objects representation in the ObjectsSpace. In most cases the value will be a domain object (e.g. a TPerson).

The Bold-aware visual controls are then in turn connected to handles. The handles can be divided into two basic groups, the root handles, and the rooted handles.

Root handles

Root handles either own or refer to an element.

BoldSystemHandle

The owner of an entire object system. Most applications will have only one system handle.
The TBoldSystemHandle is used in all examples.

BoldReferenceHandle

A handle that can be set to point at any element. It is often used as the first handle on a form showing e.g a single person.
See examples: TreeView and MultiLanguage

BoldVariableHandle

A handle that owns an element which is not an object e.g. a string, or a list of persons. The handle acts as a placeable variable which can be connected to controls, and used in variable definitions. The handle can also be used as a placeable constant through the use of InitialValues.
See examples: ComboBox and Derived Handle

BoldSQLHandle

A handle that uses the database for SQL searches. Its value is a list of objects resulting from the search. It is often used as the first handle on a form showing a list resulting from a search.
See example: SQL Handle

Rooted handles

Rooted handles derive their value from another handle. Rooted handles can be connected to each other, allowing chained evaluation.

BoldExpressionHandle

Derives its value by applying an OCL-expression to the value of its root.
See example: TreeView

BoldCursorHandle

Takes the value of its root, and interprets it as a list. It holds a CurrentIndex, and has a value which is the current index of the list. It has two properties that, in a sense, are the value of the handle List, which is the list, and Value which is the current element in the list. Multi-value controls such as grids, or the dropdown of a combobox will display the List, while single-value controls, and rooted handes will use the current element (the value property).
See examples: ComboBox and Derived Handle

BoldListHandle

This is the handle first encountered by most developers. It actually has a complex internal structure, extending a TBoldCursorHandle, containing an expressionhandle and (optionally) sorters and filters. Like the TBoldCursorHandle is has List and Value properties. It is often used for displaying an entire class (when connected to a TBoldSystemHandle), or for following n-valued associations (when connected to another handle who's value is an object).
The TBoldListHandle is used in all examples.

BoldDerivedHandle

Allows completely custom derivation of its value through user written code in an event-handler.
See example: Derived Handle
