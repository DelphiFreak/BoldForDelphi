Locking - Concurrency Management
Launch Delphi-example | Launch CBuilder-example

Note! This example demonstrates features of the ObjectSpace Synchronization Server Extension and the Concurrency Management Server Extension. License keys are required for the extensions to try the example. Use the License Manager to request evaluation keys.

To run the example, you have to register a propagator. To register a propagator you need to run the binary 'BoldPropagator.exe' (found under \bin) with the command line parameter '/regserver'.

This example shows how to use the various locking mechanisms in $(PRODUCT_BASE_NAME). It requires a propagator/lock manager server.

Some of the features shown:

  • Pessimistic locking
  • Optimistic Locking
  • Propagation
  • Exclusive database lock
  • A separate persistence server (the LockingPServer project)

How to run

Launch the LockingDemo application and click the button Start System to start the propagator. Now launch one more instance of the LockingDemo application.

Use of Concurrency Management

Points of interest:

  • The datamodules in MainDM.pas and Server/PServerDM.pas shows how to connect the various handles needed to set up pessimistic locking.
  • The TBoldModel on Common/ModelDM defines some example regions.

  • Regions (also found in [BfDInstallDir]/help/Bfd.hlp)
