ComboBox
Launch Delphi-example | Launch CBuilder-example

This example illustrates three different ways to use the BoldComboBox control. In addition to using the TBoldComboBox to set a single relation, it can also be used to set a string property by selecting a value from a stringlist or by retrieving a string from a list of objects. The latter is useful if you are content with setting a string attribute to a string retrieved from another object and don't have to be able to navigate to it.

How to run

When you open the system the main form shows two listboxes showing instances of food and school. Add a person and select additional information to that person from the comboboxes. The comboboxes School and Preferred Food shows instances from the listboxes.

The comboboxes Majors in and Favourite Music shows values from a variablehandle on the form.

The comboboxes are set up differently. To see the difference follow these instructions:

  1. Add a person and select information from the comboboxes. 
  2. Save. 
  3. Now change the names of the shool and the food you selected for the person. Use the listboxes to do that. 
  4. Save. 
  5. Select the person you created and see what's in the comboboxes for that person. The information about preferred food should consist of the new name of the food but the name of the shool should not have been updated. This is because the shool name is added as an attribute on the person and that attribute is not updated. The food information on the other hand is added to the person as a relation to the food object.

How to use BoldComboBox

To control the functionality of the comboboxes in this example we used a couple of properties in the BoldComboBox control: 

  • BoldListHandle - provides the list of elements visible in the drop down list.
  • BoldHandle - holds the handle that will be affected by changing current pointer in the drop down list.
  • BoldProperties - holds the value to be changed.
  • BoldRowProperties - properties for the elements in the list pointed to by BoldListHandle.
  • BoldSelectChangeAction - property to determine what happens when the selection changes.
  • BoldSetValueExpression - expression to use when setting the value.
     

See also

  • Programmers Reference ([BfDInstallDir]/help/Bfd.hlp) for explanation of the properties on TBoldComboBox.

Different settings

The example shows three different settings of the comboboxes.

The combobox School holds a BoldListHandle containing a list of all instances of TShool. The BoldSelectChangeAction is set to <<bdscSetText<< which indicates that when the selection changes the AsString-property of the selected element is set. Thus when you select a school in the combobox the name is added as an attribute (specified under BoldProperties) on the person and no relation is established between person and school.

The preferred food-combobox holds a BoldListHandle containing a list of all instances of TFood. In this combobox the BoldSelectChangeAction is set to <<bdcsSetValue<< and that means that when the selection changes a single-link is established from the current element of the BoldListHandle to the selected element in the combobox. The property BoldSetValueExpression holds the name of the link to update. In this example, when you change the selection of food the link preferredFood is udated and connects a person with a food-object.

The majors in- and favourite music-comboboxes both holds BoldListHandles pointing to cursorhandles. The cursorhandles in turn holds variablehandles containing string values to be shown in the comboboxes. The BoldSelectChangeAction on the comboboxes is set to <<bdcsSetText<< so when the selection changes the attribute specified under BoldProperties is updated with the selected string value.
