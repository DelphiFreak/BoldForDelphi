Derived Relations
Launch Delphi-example | Launch CBuilder-example

This example shows a code free way to implement derived associations (relations). The derivation is defined by an OCL expression in the model.

How to run

Add projects and persons to the grids by using the navigators. See what happens in the All Members and All Associates listboxes when you drag and drop persons to the Project Leader editfield and Participator listbox. When you select a person in the lower grid the listbox All Associates will show all the other persons participating in the same project(s) as the selected person. If you select a project in the upper grid you should see all members in that group in the listbox All Members. The project leader (if any) is shown in the editfield Project Leader and the other participators of the project are shown in the listbox below.

The content of the listboxes is controlled by the associations between the Person and Project class. The class Project has two associations to Person with the roles Participator and ProjectLeader, and a third, derived, association which holds allMembers for a project. Person also has a derived association to Person which holds allAssociates to a person.
