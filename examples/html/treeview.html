TreeView
Launch Delphi-example | Launch CBuilder-example

This example shows of some of the features included in the BoldTreeView control.

The following features are included in the example:

  • View a single object as root or a collection of objects.
  • Basic TreeNode handling (strictly objects and roles).
  • Enhanced TreeNode handling (manually created nodes to group objects in a multilink).
  • Using simple drag ’n drop handling to build structures.

How to run

In the main form you see five grids showing instances of mountainbike names, frames, gear models, break models and wheel models. Picked together they form a whole mountainbike. You can do that by:

  1. Select a name in the mountainbike grid. 
  2. Drag an instance of a frame to the Frame editbox. 
  3. Then drag instances of gears, breaks and wheels to the Components listbox and you have built your first mountainbike!

Expand the node with the name of the mountainbike you just built in the treeview. Under the node you will see the components of the bike.

You can also put together a bike by dragging instances from the tables and drop them on the treeview under the tab Enhanced Tree. First select a mountainbike node in the tree then drag instances from the Frames, Gears, Breakes and Wheels grids and drop them on the node. This can not be done on the treeview under the Basic Tree tab.

The button "Set Current as Root" will hide all the nodes except the top node (mountainbike) selected. To show all nodes again click the "Show All" button.

The BoldTreeView

In this example the Basic Tree shows a simple setup for node presentation. All classes have a NodeDescription to control the display of the object itself and its relations. The ListFragments are restricted to the roles available for the object.

The Enhanced Tree introduces manually created "fake" nodes, which are used to group objects in a multilink. It also includes icon control and a small example of drag & drop handling (You can drag objects to the tree to build structures).
