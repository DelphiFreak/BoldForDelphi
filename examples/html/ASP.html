ASP example
Launch Delphi-example

This example consists of a server application and a client application. ASPDemoClient is a thin client consisting of a GUI and a BoldSystem. The persistence layer resides in a remote server, ASPserver.dll, which is an ISAPI dll. The client can only communicate with the server through HTTP. 

Running the example

Installing the server

To run the example you must first install the ASPserver.dll in IIS. For more details on how to do this see the XMLDispatcher example.

Generating the database

Use the DBGenerator application to generate an interbase database for the ASP example.

Run the client

Run the ASPDemo client. Modify the ServerURL to reflect your network configuration and use the Connect button to connect to the server.

TBoldHTTPServerPersistenceHandlePassthrough and TBoldHTTPClientPersistenceHandle

The example uses a TBoldHTTPClientPersistenceHandle component on the client side and a TBoldHTTPServerPersistencehandlePassthrough component on the server side, to allow the ASPDemoClient to establish a connection with the remote peristence layer residing in ASPServer. 

ASPServer

The server defines two TWebActionItem objects, Persistence and SOAPCalls. Calls made to the persistence layer are handled by the Persistence web action. The second web action SOAPCalls shows how to use a TBoldXMLDispatcher to define custom services that could be invoke from the client.

 

Note! Please make sure to use the DBGenerator application before attempting to run the ASPDemo client for the first time.
