Simple TreeView
Launch Delphi-example

This example shows of some of the features included in the BoldTreeView control.

The following features are included in the example:

  • Build a simple treeview with node branches and leafs that corresponds to objects in the persistence layer.
  • Build simple treeview with "fake" node branches that don't corresponds to objects in the persistence layer with leafs that do.

How to run

Check the model, either the rose-model or in the model editor (double click the TBoldModel control on the form). The model of this example contains of three classes, "Company", "Department" and "Employee". The relations between them are:

  • A Company consists of several Departments
  • Each Department employs several Employees
  • A Department only belongs to one Company
  • An Employee is employed only by one Department
  • Each Employee also belongs to a Company since the Department belongs to a Company

Run the application. In the main form there are two treeview controls (the big white space that is empty before you have entered any data to the objectspace) and three grids. In the grids you enter companies, departments and employees. 

Start by entering one company and add some departments to it. For each department add some employees. As you can see the two treeviews will be updated with the objects you enter. 

In the right treeview the selected companys departments will be listed as node branches and each departments employees will be listed as node leafs. The structure follows the structure in the model. Each employee is listed under its employer (department). 

In the left treeview the structure of the model is broken. In this treeview we will list all the employees of each department of the selected company in alphabetical order taking no consideration of the departments. To do this we create "fake" node branches and with the help of OCL expressions we list the employers under the right node branch according to the employees last name.
