Model Evolution
Launch Delphi-example (Version 1) | Launch CBuilder-example (Version 1)
Launch Delphi-example (Version 2) | Launch CBuilder-example (Version 2)

This example shows how to use the Model Evolution functionality in $(PRODUCT_BASE_NAME). It consists of two versions of a simple order/entry system, where the model has been modified between the versions.

How to run

Follow the steps below to test the application:
  1. First launch version 1.
  2. Create the database and a few instances. 
  3. Save data and close version 1. 
  4. Launch version 2 and press the Evolve DB button. This will modify the database schema to support version 2 as well as version 1. The instances you created with version 1 will appear, modified to the version 2 model. You can create or modify instances from either version 1 or version 2. 
  5. Open version 1 again to see how your modifications in version 2 are transformed to version 1.

The model changes illustrated by the example includes the Person class attribute fullname from version 1 that is split into firstname and lastname in version 2. The discount attribute of the Order class in version 1 is moved to a new associated class Discount in version 2. The cost attribute (intended to be in swedish kronor) of the Product class in version 1 is replaced with costEuro (in Euro). in version 2

How to evolve models

Points of interest in this example:

  • The BoldObjectUpgraderHandle on the datamodule in dPersistence in version 2 is responsible for performing the on-the-fly upgrading of objects when read in version 2.
  • The file examples/Compound/Version2/ModelEvClasses.inc contains the methods for ensuring backward compatibility (TDiscount.ReceiveQueryFromOwned, TPerson.PrepareUpdate, TProduct.PrepareUpdate) as well as the acutal upgrading logic (TOrder.Upgrade, TPerson.Upgrade, TProduct.Upgrade).
