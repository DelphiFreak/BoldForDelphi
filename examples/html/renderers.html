Renderers
Launch Delphi-example | Launch CBuilder-example

This example illustrates how to use the renderers included in Bold for Delphi. In this example one use of the renderer is to set or split a full name into first name and last name. A second use is to have the renderer to change the color of a value.

How to run

Add new persons to the grid using the navigator at the bottom of the main form. The persons that are created are automatically given a first name and a last name at instantiation. The column Full Name shows the first name and the last name concatenated using a renderer as described below. See what happens if you change the first name. The full name is updated according to the change. Now change the value of the full name. The changes applies on the first name and last name respectively.

Each person created is also given a salary. The color of the value is changed if the salary is less than given salary breakpoint entered in the editbox at the bottom of the form. Enter a number in the editbox and see what happens in the salary-column for those persons with a salary below that number.

Use of renderers

There are two ways to implement derived values in Bold. First of all you have the opportunity to use OCL expressions to define the derived attribute inside the model. This has the advantage of minimizing the amount of code necessary. In some cases you will find the need to use code to accomplish the task at hand. One way to do that is to use renderers. With renderers you have the capability of not only writing code for derived attributes but also to implement reverse derivation.

OCL derivation vs. renderers

In this example we use objects of a class Person with the attributes FirstName, LastName and Salary. If we want to present an object of class Person with his or her full name we can accomplish this by simply creating a derived attribute /FullName and assigning it the OCL expression <<FirstName + ' ' + LastName<<. This will give us an attribute that is write protected within the user interface.

If we want to be able to edit the full name and have, for example, the string "Paul Smith" split into a first name value of "Paul" and a last name value of "Smith" we need to use a renderer.

In this example we use two instances of the renderer BoldAsStringRenderer. One instance implements the OnGetAsString and OnSetAsString methods to retrieve and edit the full name of a person. The renderer merges the FirstName and LastName attributes with a single space between them to form the full name. If this full name is edited the set-method splits the string into first name and last name and stores the values in the two attributes. The other renderer is a small example of implementing the OnSetColor method.
